# 😵 社死角落 — 把一个人的尴尬，变成一群人的快乐

> "每个人心里都有一段不敢开口的社死往事。这里，是它们安全着陆的地方。"

## 一句话介绍

**社死角落**是一个 AI 驱动的匿名尴尬经历共鸣社区——让用户零压力地倾倒那些"说不出口"的社死故事，在陌生人的哈哈哈里获得治愈，顺便被 AI 盖上一个"地狱级社死"的趣味认证章。

---

## 我们解决什么问题

每个人都经历过"恨不得原地消失"的时刻：把老板的朋友圈评论"说的太对了"手滑打成"说的太多了"、在会议室以为没人放了个炸天屁结果同事就坐在角落、对着镜子做了三秒钟的"你先请"……

**这些经历有一个共同的命运：被死死憋在心里。**

- 发朋友圈？社交形象崩塌。
- 跟朋友说？开不了口。
- 写日记？没有共鸣等于白写。

但恰恰是这些最真实的尴尬，最能让人笑出来、最能引发"我也是"的共鸣。

**人们缺的不是勇气，而是一个没有身份负担的角落。**

社死角落就是这个角落——匿名、轻量、零门槛。把你的尴尬扔进来，让全世界陪你一起笑。

---

## AI 解决方案

### 核心理念：AI 不是来打分的，是来陪你玩的

我们没有把 AI 做成冷冰冰的内容审核工具，而是让它成为一个"社死评审官"——用幽默的方式帮你的故事贴上一枚趣味认证章。

### 三维度 AI 社死评级引擎

接入 GLM 大语言模型（`glm-4-flash`），从三个维度实时分析每段经历：

| 维度 | 评估内容 | 举例 |
|------|---------|------|
| **尴尬烈度** | 当事人想挖地洞的程度 | 普通口误 vs 全公司直播翻车 |
| **波及范围** | 多少人目睹了你的社死 | 只有自己知道 vs 300人视频会议 |
| **不可挽回度** | 能不能圆回来 | 说错一个字 vs 永久社交性死亡 |

AI 综合三维度输出 **1-100 的社死指数** + **五级趣味评级**：

```
90-100  🔥 地狱级社死    ← "建议搬家"
75-89   😱 极高危社死    ← "半年内别出门"
60-74   😰 高危社死      ← "一周缓不过来"
40-59   😅 中度社死      ← "当时很尬，事后能笑"
1-39    🙈 轻度社死      ← "小场面，不值一提"
```

### 实时评分体验

用户写故事时，**输入 15 个字即触发实时评级预览**（1200ms 防抖），边写边看自己的社死等级跳动——这种即时反馈让投稿变成了一场"看看我能多社死"的游戏，极大降低了分享的心理门槛。

### 智能降级保障

当 AI 接口不可用时，自动切换到**本地关键词权重评分**（如"领导""全班""直播"等高社死权重词），确保应用永不掉线。

## 产品功能

### 📱 五个核心页面

#### 1. 首页 · 社死排行榜 (`index.html`)

三个内容分发维度，让好故事被看见：

- 🔥 **热门** — 点赞最多的社死故事，社交验证驱动
- 🆕 **最新** — 实时更新的投稿流，FOMO（错过焦虑）驱动
- 👑 **名人堂** — 社死指数最高的传奇故事，成就驱动

每张卡片展示：随机 Emoji 头像、社死指数、点赞数、评论数、分类标签、排名徽章。

#### 2. 详情页 (`detail.html`)

- 完整故事 + 社死指数动画进度条
- AI 评级详情（等级 + 原因）
- 点赞 / 评论 / 分享 / 收藏
- **内联社死证书卡片**（异步加载，不阻塞主内容）

#### 3. 投稿页 (`submit.html`)

- 500 字文本框 + 实时字数统计
- **AI 实时评分预览**（核心体验亮点）
- 9 类标签选择（职场 / 恋爱 / 校园 / 生活 / 网络 / 家庭 / 购物 / 医院 / 其他）
- 匿名开关（默认开启）+ 城市选择弹窗
- 草稿自动保存（localStorage）
- 支持编辑已发布内容（`?edit=<id>` 模式）

#### 4. 我的页面 (`mine.html`)

- 个人社死经历管理（按社死指数排序）
- 编辑 / 删除（两步确认 + 淡出动画）
- 空状态引导投稿

#### 5. 社死等级证书 (`certificate.html`)

- 暗色主题的仪式感证书页面
- 展示：社死等级徽章、最高指数、故事数、全站排名、代表作
- 保存图片 / 社交分享
- **把尴尬变成一种社交货币** —— "我是地狱级社死王"反而成了一种骄傲

        ┌────────────┴────────────────┐
        ▼                             ▼
┌───────────────┐           ┌──────────────────┐
│   Supabase    │           │   GLM-4-Flash    │
│  PostgreSQL   │           │   (智谱 AI)       │
│  + RLS + RPC  │           │  社死指数评估      │
└───────────────┘           └──────────────────┘
        │
        │ 降级
        ▼
┌───────────────┐           ┌──────────────────┐
│  localStorage │           │  本地关键词评分    │
│  (离线兜底)    │           │   (AI 降级方案)   │

### 技术选型

| 层级 | 技术 | 选型理由 |
|------|------|---------|
| 前端 | Vanilla JS + HTML/CSS | 零依赖、秒开、无构建成本，纯静态部署 |
| 数据库 | Supabase (PostgreSQL) | 免费托管、REST API 开箱即用、行级安全策略 |
| AI 模型 | GLM-4-Flash | 中文语境理解强、响应速度快、成本低 |
| 安全 | 原生 fetch + XSS 转义 | 规避 CSP eval 问题，`escHtml()` 防注入 |
| 持久化 | localStorage | 点赞去重、草稿恢复、离线降级 |

### 数据模型

```sql
posts:    id · content · tag · death_index · death_level · death_reason
          anonymous · location · likes · author_id · created_at

comments: id · post_id · content · author_id · created_at
```

- `toggle_like()` SQL 函数实现原子点赞（防并发冲突）
- 行级安全策略（RLS）控制公开读写
- 级联删除：帖子删除时自动清理评论

## 项目亮点

### 1. "越尴尬越想写完"的产品心理学

实时 AI 评分不是功能，是一个**行为激励钩子**：用户写了 15 个字看到"轻度社死"，好胜心驱动他继续补充细节，直到评级跳到"地狱级社死"——**AI 评分越高，成就感越强，故事细节越丰富**。

### 2. 匿名设计消除分享焦虑

- 默认匿名 + 随机 Emoji 头像（基于 author_id 哈希确定性生成）
- 无需注册、无需登录、无需绑定手机号
- 用户标识仅存 localStorage，关闭浏览器即可"重新做人"

### 3. 把尴尬转化为社交货币

社死证书系统将负面情绪包装为**可炫耀的成就**。"我是地狱级社死王"听起来是自嘲，实际是一种稀缺身份标签——越社死越骄傲，越骄傲越想分享。

### 4. 双层降级保障，永不掉线

```
AI 在线 → GLM 大模型实时评分
AI 离线 → 本地关键词权重评分
DB 在线 → Supabase 云端持久化
DB 离线 → localStorage 本地存储 + 种子数据
```

任何一层挂掉，用户体验均不受损。

### 5. 极致轻量

- 全站 **0 个 npm 依赖**（前端）
- 无 React / Vue / Webpack / Vite
- 5 个 HTML 文件 + 1 个 JS 文件 = 完整应用
- 打开即用，首屏加载 < 1s

## 快速开始

### 1. 克隆项目

```bash
git clone <repo-url>
cd 0225-shesi
```

### 2. 初始化数据库

在 [Supabase Dashboard](https://app.supabase.com) > SQL Editor 中执行：

```bash
# 查看 SQL schema
cat schema.sql
```

### 3. 导入种子数据（可选）

```bash
npm install
node migrate.mjs
```

### 4. 启动

```bash
# 任意静态服务器即可
npx serve .
# 或
python3 -m http.server 8000
```

打开 `http://localhost:8000` 即可使用。
